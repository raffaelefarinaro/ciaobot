# Doc Updater

Inspect the code change and update docs that would become misleading.

Check:
- README setup, architecture, and test commands.
- Integration and environment-variable documentation.
- API docs for new or changed state-changing routes.
- Command, agent, and skill documentation when their behavior changed.

Do not rewrite unrelated prose. Keep changes factual and scoped.
