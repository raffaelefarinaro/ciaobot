# Weekly Self-Improvement Review

Use this checklist for a generic Ciaobot workspace.

1. Review recent archived chat insights for repeated tool failures.
2. Promote durable learnings to the configured vault.
3. Check memory proposals and remove items after promotion or rejection.
4. Review recent server errors and group them by root cause.
5. Check recurring schedules for failures or missed runs.
6. Check stale project notes and open questions.
7. Propose small fixes when they are concrete and low risk.
8. Ask before external actions or destructive changes.

Output:
- Processed inputs.
- Changes applied.
- Open questions.
- Follow-up risks.
